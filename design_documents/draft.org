* slurm client

Just like the PBS client, a slurm client is important.

https://doc.hpc.ifremer.fr/compute/slurm/slurm-rest/

The steps from connection to displaying are:
- ssh connect, using asyncssh
- call `scontrol token lifespan=900`, note the token together with the call time
- detect latest slurm version using the /openapi/v3 endpoint
- use httpx to perform the actual queries

** TUI
The TUI should have:
- ping status indicator
- users info
  - account list / account details
  - association list / details
  - instance list / details
  - user list / details
  - wckey?
  - qos?
- queue list
- queue summary tables
- job list
- job information
  - state / resources / etc
  - environment variables
  - log fields (which should be self-updating like `tail -f` and have builtin filters, e.g. for filtering / interpreting escape sequences)
- node information
- partition information
- reservations
- shares
